5G Registration Call Flow

Updated: Apr 19, 2026 Live · 3GPP Spec 📖 8 min read

Overview

The 5G Registration procedure is the fundamental process by which a User Equipment (UE) attaches to the 5G network and registers its presence with the 5G Core (5GC). This procedure establishes the UE's identity, performs mutual authentication between the UE and network, activates NAS security, and creates mobility management context in the Access and Mobility Management Function (AMF). Registration is mandatory before the UE can access any 5G services, establish PDU sessions, or receive mobile-terminated services.

This procedure involves multiple 5G Core network functions including the AMF for mobility management, the Authentication Server Function (AUSF) for authentication, and the Unified Data Management (UDM) for subscriber data retrieval. The registration flow encompasses NAS signaling over the N1 interface, NGAP signaling over the N2 interface between gNB and AMF, and service-based interface interactions among core network functions to complete authentication, security activation, and subscriber registration.

  • Authenticate the UE using 5G AKA or EAP-AKA' and verify subscriber credentials
  • Establish NAS security context with selected ciphering and integrity protection algorithms
  • Assign 5G-GUTI, allowed network slices (NSSAI), and tracking area list for mobility management
  • Create UE context in AMF and update UDM with current serving AMF and registration state

System Architecture

UE ⇄ gNB ⇄ AMF ⇄ AUSF ⇄ UDM
5G Core Network Reference Architecture (3GPP TS 23.501)

Procedure Steps

1
UE sends Registration Request
The UE initiates registration by sending a NAS Registration Request message containing the Subscription Concealed Identifier (SUCI), registration type (initial, mobility, periodic, emergency), UE security capabilities (supported encryption and integrity algorithms), requested NSSAI for network slice selection, and 5GMM capability information.
2
gNB forwards via NGAP Initial UE Message
The gNB encapsulates the NAS Registration Request in an NGAP Initial UE Message and forwards it to the AMF over the N2 interface. The NGAP message includes the selected PLMN identity, user location information (cell ID), RRC establishment cause, and the transparent NAS PDU container.
3
AMF selects AUSF and initiates authentication
The AMF decrypts the SUCI to obtain the SUPI (Subscription Permanent Identifier), selects an appropriate AUSF, and invokes the Nausf_UEAuthentication_Authenticate service operation. The AUSF coordinates with UDM via Nudm_UEAuthentication_Get to retrieve authentication vectors (5G HE AV containing RAND, AUTN, XRES*, KAUSF).
4
AMF sends Authentication Request to UE
The AMF sends a NAS Authentication Request message to the UE containing the RAND (random challenge), AUTN (authentication token for network authentication), ngKSI (key set identifier), and ABBA parameter (anti-bidding down between architectures) to prevent downgrade attacks.
5
UE responds with Authentication Response
The UE validates AUTN using its USIM to authenticate the network, computes RES* from RAND using the K key stored in USIM, and sends a NAS Authentication Response containing RES* back to the AMF. The AMF verifies RES* against HRES* derived from XRES* to authenticate the UE.
6
AMF sends Security Mode Command
Upon successful authentication, the AMF derives the KAMF key and sends a NAS Security Mode Command to the UE. This message specifies the selected NAS ciphering algorithm (NEA0, NEA1, NEA2, NEA3), NAS integrity algorithm (NIA1, NIA2, NIA3), ngKSI, and replayed UE security capabilities to prevent bidding-down attacks.
7
UE sends Security Mode Complete
The UE derives the same KAMF key, generates NAS encryption (KNASenc) and integrity (KNASint) keys, activates the security context, and sends a NAS Security Mode Complete message. This message is integrity-protected and ciphered using the newly activated security context, and may include the IMEISV (International Mobile Equipment Identity Software Version).
8
AMF sends Initial Context Setup Request to gNB
The AMF sends an NGAP Initial Context Setup Request to the gNB to establish the UE context in the RAN. This message includes the allowed NSSAI, UE security capabilities, security key, UE aggregate maximum bit rate (AMBR), and may include PDU session resource setup requests if default bearers are being established.
9
AMF sends Registration Accept to UE
The AMF sends a NAS Registration Accept message to the UE containing the assigned 5G-GUTI (Globally Unique Temporary Identifier), allowed NSSAI (network slices the UE is permitted to access), TAI list (tracking areas where UE can roam without re-registration), mobility restrictions, periodic registration timer (T3512), and configured NSSAI.
10
UE sends Registration Complete
The UE acknowledges successful registration by sending a NAS Registration Complete message to the AMF. The UE stores the 5G-GUTI, enters 5GMM-REGISTERED state, starts the periodic registration timer T3512, and is now ready to establish PDU sessions and access 5G services.
11
AMF updates UDM via Nudm_UECM_Registration
The AMF invokes the Nudm_UECM_Registration service operation toward the UDM over the N8 interface to register itself as the serving AMF for this UE. The UDM stores the AMF address, registration timestamp, and UE context information, completing the registration procedure and enabling mobile-terminated services.

Message Field Details

Registration Request
5GS Registration Type — Indicates type: Initial registration (001), Mobility registration updating (010), Periodic registration updating (011), Emergency registration (100), or SNPN onboarding (101)
ngKSI (NAS Key Set Identifier) — Identifies the NAS security context (0-6 for native context, 7 for no key available). Used to reference existing security keys if available.
5GS Mobile Identity — Contains SUCI (Subscription Concealed Identifier) for initial registration, or 5G-GUTI for mobility/periodic updates. SUCI is encrypted SUPI using home network public key.
UE Security Capability — Lists supported encryption algorithms (5G-EA0/NEA0, 128-NEA1, 128-NEA2, 128-NEA3) and integrity algorithms (5G-IA0/NIA0, 128-NIA1, 128-NIA2, 128-NIA3) in preference order
Requested NSSAI — Network Slice Selection Assistance Information containing S-NSSAI(s) (Slice/Service Type + Slice Differentiator) that UE wants to access
5GMM Capability — UE's 5G mobility management capabilities including support for features like LPP, HO attach, S1 mode, N1 mode, and restriction of use of enhanced coverage
UE Status — Optional field indicating N1 mode, S1 mode capability, and EMM/5GMM state for interworking scenarios
MICO Indication — Mobile Initiated Connection Only mode indication - if UE prefers to be reachable only when it initiates connections to save power
Last Visited Registered TAI — Tracking Area Identity where UE was last registered, helps AMF with context retrieval from old AMF
Authentication Request
ngKSI — NAS Key Set Identifier assigned by the network to identify this authentication and key agreement session (0-6)
ABBA (Anti-Bidding down Between Architectures) — 2-octet parameter binding authentication to the serving network type to prevent bidding-down from 5G to EPS. Contains length indicator.
RAND (Random Challenge) — 128-bit random challenge generated by AUSF/UDM, used by UE USIM to compute authentication response and derive keys
AUTN (Authentication Token) — 128-bit token (SQN⊕AK || AMF || MAC) allowing UE to authenticate the network and verify freshness of authentication vector
EAP Message — Optional field used when EAP-AKA' authentication method is employed instead of 5G AKA
Authentication Response
Authentication Response Parameter (RES*) — 128-bit response computed by UE USIM as RES* = KDF(CK||IK, serving network name, RAND, RES). AMF verifies against HRES* = KDF(KAUSF, RAND, RES*)
EAP Message — Optional field containing EAP response when EAP-AKA' is used for authentication
Security Mode Command
Selected NAS Security Algorithms — Type of ciphering algorithm (5G-EA0, 128-5G-EA1, 128-5G-EA2, 128-5G-EA3) and integrity algorithm (5G-IA0, 128-5G-IA1, 128-5G-IA2, 128-5G-IA3) selected by AMF
ngKSI — NAS Key Set Identifier (0-6) referencing the KAMF and associated security context to be used
Replayed UE Security Capabilities — Echo of UE security capabilities from Registration Request, allowing UE to verify no bidding-down attack occurred during signaling
IMEISV Request — Optional flag requesting UE to include its IMEISV (International Mobile Equipment Identity Software Version) in Security Mode Complete
Additional 5G Security Information — Optional field indicating whether horizontal derivation (KAMF to KAMF') or vertical derivation is required for subsequent security contexts
EAP Message — Optional EAP success message when EAP-AKA' authentication is used
Security Mode Complete
IMEISV (Mobile Identity) — International Mobile Equipment Identity and Software Version Number (16 digits), included if requested by network in Security Mode Command
NAS Message Container — Optional container carrying a previously failed NAS message that can now be sent with integrity protection and encryption activated
Registration Accept
5GS Registration Result — Indicates result: 3GPP access, Non-3GPP access, or both. Also indicates if SMS over NAS is allowed and whether MICO mode is approved.
5G-GUTI — Globally Unique Temporary Identifier assigned to UE: . Used for subsequent registrations.
Allowed NSSAI — List of S-NSSAIs (Single Network Slice Selection Assistance Information) the UE is allowed to use in the current registration area
TAI List — List of Tracking Area Identities where UE can roam without re-registration. Each TAI contains PLMN ID and TAC (Tracking Area Code).
Configured NSSAI — Network slices configured for the UE for the current PLMN, valid across registration areas
T3512 Value (Periodic Registration Timer) — Timer duration (in minutes or hours) for periodic registration update. UE must re-register before timer expires to remain registered.
T3502 Value (Registration Retry Timer) — Timer value instructing UE how long to wait before retrying registration if procedure fails
Emergency Number List — Optional list of emergency numbers (like 112, 911) valid in current country/PLMN
5GS Network Feature Support — Indicates network support for features like IMS voice over PS session, emergency services, location services, and UP integrity protection
LADN Information — Local Area Data Network information containing DNN and associated TAI lists where LADN is available
MICO Indication — Indicates whether AMF has accepted UE's MICO mode request for power-saving
Service Area Restrictions — Optional restrictions or allowed service areas for the UE subscription
Registration Complete
SOR Transparent Container — Optional Steering of Roaming acknowledgment container when operator policies have been updated
