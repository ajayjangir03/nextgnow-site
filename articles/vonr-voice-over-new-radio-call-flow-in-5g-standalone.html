VoNR (Voice over New Radio) Call Flow in 5G Standalone

Updated: Apr 19, 2026 Live · 3GPP Spec 📖 8 min read

Overview

Voice over New Radio (VoNR) is the native voice solution for 5G standalone (SA) networks, delivering voice services over the 5G packet core using IMS (IP Multimedia Subsystem). Unlike VoLTE which relies on LTE/EPC infrastructure, VoNR uses 5G NR radio access and 5G Core network functions to establish end-to-end QoS-guaranteed voice calls with dynamic PDU session modification. The call flow involves SIP signaling over the IMS domain, coordinated with SMF-driven QoS flow establishment to ensure guaranteed bit rate and low latency for the voice media stream.

VoNR represents a critical milestone in 5G commercialization, enabling operators to decommission legacy circuit-switched infrastructure and offer all services over a unified 5G packet network. It leverages network slicing, dedicated QoS flows with 5QI=1 for voice, and tight integration between IMS P-CSCF and 5G Core PCF for policy enforcement, ensuring carrier-grade voice quality while supporting advanced features like video calling, real-time text, and seamless handover to EPS Fallback when needed.

  • Establish IMS registration over 5G PDU session to enable SIP-based voice services
  • Trigger PDU session modification to create dedicated QoS flow with 5QI=1 for guaranteed voice bearer
  • Coordinate SIP INVITE signaling with dynamic QoS provisioning via PCF and SMF for end-to-end call setup

Network Functions Involved

UE User Equipment with IMS client for SIP signaling and voice codec
gNB 5G Base Station — manages radio resources and QoS flow mapping to DRBs
AMF Access and Mobility Management Function — handles NAS signaling and mobility
SMF Session Management Function — manages PDU session modification and QoS flow establishment
UPF User Plane Function — packet routing and QoS enforcement for voice media
PCF Policy Control Function — provides PCC rules for voice QoS flow (5QI=1, GBR)
P-CSCF Proxy Call Session Control Function — IMS entry point, SIP proxy for UE
S-CSCF Serving CSCF — IMS session control, routing SIP messages, service invocation
I-CSCF Interrogating CSCF — queries HSS/UDM to locate S-CSCF for called party
HSS/UDM Home Subscriber Server / Unified Data Management — IMS subscription and routing data

System Architecture

UE ⇄ gNB ⇄ AMF | UE ⇄ UPF ⇄ P-CSCF ⇄ S-CSCF ⇄ I-CSCF | SMF ⇄ PCF ⇄ UPF
5G System with IMS Integration (3GPP TS 23.501, TS 23.228, TS 23.502)

Procedure Steps

1
UE establishes default PDU session for IMS
UE → SMF N1/N11
The UE initiates a PDU Session Establishment Request for the IMS DNN (Data Network Name), typically using a dedicated IMS slice. The SMF coordinates with PCF to obtain default QoS rules and establishes a PDU session with default QoS flow (5QI=9, non-GBR) for IMS signaling. The UPF is configured to route IMS traffic to the P-CSCF.
DNN=imsS-NSSAI (IMS slice)PDU Session Type=IPv4v6SSC Mode=1
2
UE performs IMS Registration (SIP REGISTER)
UE → P-CSCF User Plane (N6)
The UE IMS client sends a SIP REGISTER message over the default PDU session to the P-CSCF. The P-CSCF forwards it to I-CSCF, which queries HSS/UDM to locate the assigned S-CSCF. IMS AKA authentication is performed, and upon successful registration, the UE receives a 200 OK response with Service-Route headers, establishing the IMS signaling path.
SIP URI (sip:user@domain)Public User IdentityContactAuthorization
3
Calling UE initiates voice call (SIP INVITE)
UE-A → P-CSCF User Plane (N6)
The calling UE (UE-A) sends a SIP INVITE with Session Description Protocol (SDP) offering voice codecs (typically AMR-WB for VoNR) and QoS preconditions. The P-CSCF inspects P-Access-Network-Info to identify 5G access and forwards the INVITE to S-CSCF. The S-CSCF applies originating services and routes the INVITE toward the terminating party via I-CSCF.
SIP INVITESDP (codec offer: AMR-WB)P-Access-Network-InfoRequire: precondition
4
P-CSCF triggers policy update for QoS
P-CSCF → PCF N5 (via Rx)
The originating P-CSCF extracts SDP media information from the SIP INVITE and sends an AAR (AA-Request) to the PCF over the Rx-equivalent interface (N5). The PCF generates PCC rules defining a dedicated QoS flow with 5QI=1 (GBR for conversational voice), guaranteed bit rate, and packet delay budget of 100ms. Flow status is initially set to Enabled-Downlink to implement resource reservation preconditions.
Media Component (audio)Codec=AMR-WBBandwidth=24 kbps UL/DLFlow Status=Enabled-Downlink
5
PCF provisions PCC rules to SMF
PCF → SMF N7
The PCF sends an N7 session policy update to the SMF containing the PCC rules for the voice bearer. These rules specify 5QI=1 (conversational voice), guaranteed bit rate, priority level, and packet delay budget. The SMF prepares to modify the existing PDU session by creating a new dedicated QoS flow for voice media while maintaining the default QoS flow for SIP signaling.
5QI=1GBR=24 kbps UL/DLPriority Level=20Packet Delay Budget=100ms
6
SMF initiates PDU Session Modification
SMF → UE N1 (via AMF)
The SMF triggers a PDU Session Modification procedure by sending N1 SM messages via the AMF to the UE. The modification includes the new QoS flow parameters, packet filters to match voice RTP traffic, and QoS rules. The SMF also configures the UPF with corresponding N4 rules to enforce the new QoS flow and perform appropriate packet marking and policing.
QoS Flow ID (QFI)5QI=1GBR parametersQoS RulesQoS Flow Description
7
gNB establishes Dedicated Radio Bearer (DRB)
gNB → UE Radio (NR)
The gNB receives the QoS profile for the new QoS flow from the AMF and establishes a dedicated radio bearer (DRB) optimized for voice traffic. The DRB is configured with acknowledged RLC mode for reliability, appropriate PDCP settings for header compression, and high scheduling priority. The gNB maps the QoS flow (identified by QFI) to this DRB and performs RRC reconfiguration with the UE.
DRB IDRLC mode=AcknowledgedPDCP configScheduling priorityLogical channel priority
8
UE sends SIP 183 Session Progress
UE-A → P-CSCF User Plane (N6)
After the QoS flow is successfully established, the calling UE sends a SIP 183 Session Progress response with an SDP answer containing the selected codec and media parameters. This indicates that local resource reservation (QoS flow setup) is complete but the call is not yet fully established. The message is forwarded through the IMS core toward the terminating party.
183 Session ProgressSDP answerRequire: preconditionP-Early-Media
9
Terminating UE receives SIP INVITE
S-CSCF → UE-B User Plane (N6)
The S-CSCF serving the terminating UE (UE-B) forwards the SIP INVITE through the terminating P-CSCF to UE-B. The terminating P-CSCF similarly triggers QoS resource allocation by contacting its PCF, which provisions PCC rules to the terminating SMF. The terminating UE's PDU session is also modified to create a dedicated voice QoS flow.
SIP INVITESDP offerP-Asserted-IdentityAlert-Info
10
Terminating UE alerts and establishes QoS
UE-B → P-CSCF User Plane (N6)
Once the terminating UE's dedicated QoS flow is established, UE-B begins alerting the user (ringing) and sends a SIP 180 Ringing response back through the IMS core. This response includes an SDP answer confirming codec selection and indicates that terminating-side QoS preconditions are satisfied. The response propagates back to the originating UE via the signaling path.
180 RingingSDP answerQoS preconditions met
11
P-CSCF updates flow status to bidirectional
P-CSCF → PCF N5 (via Rx)
When both originating and terminating preconditions are met (indicated by SIP signaling progress), the P-CSCFs update the PCF to change the flow status from Enabled-Downlink to Enabled (bidirectional). The PCF updates the PCC rules accordingly and pushes them to the SMF, which reconfigures the UPF to allow bidirectional RTP traffic flow for the voice call.
Flow Status=EnabledMedia authorization
12
Called party answers (SIP 200 OK)
UE-B → UE-A User Plane (N6) via IMS
When the called party accepts the call, UE-B sends a SIP 200 OK response with the final SDP answer confirming the negotiated codec (AMR-WB), media IP addresses, and RTP port numbers. This message is routed through the terminating P-CSCF, S-CSCF, and originating P-CSCF back to UE-A, signaling successful call establishment.
200 OKFinal SDP answerContactSession parameters
13
Calling party sends SIP ACK
UE-A → UE-B User Plane (N6) via IMS
UE-A acknowledges the 200 OK by sending a SIP ACK message, completing the three-way SIP handshake for session establishment. The ACK is routed using the established dialog route (typically bypassing the S-CSCF and going directly through P-CSCFs for optimized routing). At this point, the SIP session is fully established.
ACKRoute headers
14
Bidirectional RTP voice media flow begins
UE-A → UE-B User Plane (N3/N9/N6)
With the SIP session established and QoS flows active on both sides, the UEs begin exchanging RTP voice packets containing AMR-WB encoded audio. Voice packets traverse the gNB (mapped to high-priority DRB), through the UPF with 5QI=1 QoS enforcement, and across the N9/N6 interfaces. Both UPFs enforce the guaranteed bit rate, priority scheduling, and packet delay budget to ensure carrier-grade voice quality. RTCP packets provide QoS feedback.
RTP payload=AMR-WBRTCP5QI=1 QoS enforcementPacket marking
15
Call termination (SIP BYE)
UE-A → UE-B User Plane (N6) via IMS
When either party ends the call, the terminating UE sends a SIP BYE message through the IMS core to the other party. The P-CSCFs detect the session termination and trigger resource release by sending session termination notifications to their respective PCFs over the N5 interface.
BYEReason
16
PCF triggers QoS flow removal
PCF → SMF N7
The PCF sends N7 policy updates to both originating and terminating SMFs to remove the PCC rules associated with the voice call. The SMF initiates PDU session modification procedures to delete the dedicated QoS flows (5QI=1) while maintaining the default QoS flow for ongoing IMS signaling.
PCC rule removalQoS flow release
17
SMF releases dedicated QoS flows
SMF → UE N1 (via AMF) / N4
The SMF sends PDU Session Modification commands to the UE (via N1 NAS messaging) and to the UPF (via N4) to remove the dedicated voice QoS flows. The gNB releases the corresponding dedicated radio bearers, freeing up radio resources. The UE acknowledges the modification, and the PDU session returns to its default state with only the IMS signaling QoS flow remaining active.
QoS Flow deletionDRB release
18
SIP 200 OK to BYE completes call release
UE-B → UE-A User Plane (N6) via IMS
The called party responds to the BYE with a SIP 200 OK, confirming session termination. Both UEs stop the RTP media streams, and the IMS session is fully released. The P-CSCFs complete the resource cleanup by confirming the session termination to the PCFs, and all call-related state is cleared from the IMS and 5G Core network functions. The PDU sessions remain active for potential future IMS services.
200 OK
